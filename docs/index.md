# ScyllaDB Provider

Use the ScyllaDB provider to manage access control in a [ScyllaDB](https://www.scylladb.com/) cluster,
including roles, keyspace grants, and table-level permissions. The provider supports both password-based
and TLS client certificate authentication. The provider also supports proxy connections when `HTTP_PROXY`
or `HTTPS_PROXY` environment variables are set. Note that this provider does not manage the lifecycle
of ScyllaDB clusters or keyspaces. It is focused on access control management.

## Example Usage

```terraform
# Authentication with username and password
provider "scylladb" {
  host = "localhost:9042"
  auth_login_userpass {
    username = "cassandra"
    password = "cassandra"
  }
}
```

## Environment Variables

The following environment variables are supported. They are read first; an explicit provider attribute overrides them.

| Variable | Provider Attribute | Description |
|---|---|---|
| `SCYLLADB_HOST` | `host` | Hostname or IP address of the ScyllaDB instance |
| `SCYLLADB_PASSWORD` | `auth_login_userpass.password` | Password for username/password authentication |
| `SCYLLADB_CA_CERT` | — | PEM-encoded CA certificate (alternative to `ca_cert_file`) |
| `SCYLLADB_CLIENT_CERT` | — | PEM-encoded client certificate (alternative to `auth_tls.cert_file`) |
| `SCYLLADB_CLIENT_KEY` | — | PEM-encoded client private key (alternative to `auth_tls.key_file`) |

## Schema

### Optional

- `auth_login_userpass` (Block, Optional) Login to ScyllaDB using the userpass method (see [below for nested schema](#nestedblock--auth_login_userpass))
- `auth_tls` (Block, Optional) Login to ScyllaDB using TLS (see [below for nested schema](#nestedblock--auth_tls))
- `ca_cert_file` (String) Path to the CA certificate file for TLS connections.
- `host` (String) Hostname or IP address of the ScyllaDB instance with a port if necessary. e.g. localhost:9042
- `skip_host_verification` (Boolean) Skip TLS host verification. Default is `false`.
- `system_auth_keyspace` (String) The keyspace where ScyllaDB stores authentication and authorization information. Default is `system_auth`.
