# Resource scylladb_grant

Manages a [privilege grant](https://docs.scylladb.com/stable/operating-scylla/security/authorization.html#grant-permission)
on a ScyllaDB resource.

Grant resources are idempotent: applying the same grant multiple times does not produce an error.
If a grant is modified outside of Terraform, the resource is automatically marked for replacement
on the next plan.

Please note that this resource should not be used with `scylladb_keyspace_grants` or `scylladb_table_grants`
since it may update grants on the same keyspaces/tables and it will cause conflicts.

## Example Usage

```terraform
resource "scylladb_role" "admin" {
  role         = "admin"
  can_login    = false
  is_superuser = false
}

resource "scylladb_grant" "admin_alter_cycling" {
  role_name     = scylladb_role.admin.role
  privilege     = "ALTER"
  resource_type = "KEYSPACE"
  keyspace      = "cycling"
}
```

## Schema

### Required

- `privilege` (String) The privilege to grant.
- `resource_type` (String) The type of resource (e.g., ALL KEYSPACES, KEYSPACE, TABLE).
- `role_name` (String) The role to which the privilege is granted.

### Optional

- `identifier` (String) The identifier of the resource (e.g., table name).
- `keyspace` (String) The keyspace of the resource.

### Read-Only

- `id` (String) The ID of the grant.
- `last_updated` (String) The timestamp of the last update to the grant.
- `permissions` (List of String) The recorded permission for the grant

## Supported Values

### `privilege`

| Value | Description |
|---|---|
| `ALL PERMISSIONS` | Grants all applicable permissions on the resource |
| `ALTER` | Allows altering keyspace or table schema |
| `AUTHORIZE` | Allows granting or revoking permissions |
| `CREATE` | Allows creating keyspaces or tables |
| `DESCRIBE` | Allows listing objects |
| `DROP` | Allows dropping keyspaces or tables |
| `MODIFY` | Allows `INSERT`, `UPDATE`, `DELETE`, and `TRUNCATE` |
| `SELECT` | Allows reading data with `SELECT` |

### `resource_type`

| Value | Required Additional Attributes |
|---|---|
| `ALL KEYSPACES` | — |
| `KEYSPACE` | `keyspace` |
| `TABLE` | `keyspace`, `identifier` (table name) |

Note that `ALL USERS` and `USER` resource types are not supported by this provider,
as the provider focuses on access control management for keyspaces and tables, and
does not manage user accounts or roles directly.

## Import

Grants can be imported using the pipe-delimited ID format
`RoleName|Privilege|ResourceType|Keyspace|Identifier`. Leave `Keyspace` and
`Identifier` empty when they do not apply.

```shell
# Import a Grant resource for a specific table with SELECT permission
terraform import scylladb_grant.example "admin|SELECT|TABLE|cycling|races"

# Import a Grant resource for a keyspace with ALTER permission
terraform import scylladb_grant.example "admin|ALTER|KEYSPACE|cycling|"

# Import a Grant resource for all keyspaces with MODIFY permission
terraform import scylladb_grant.example "admin|MODIFY|ALL KEYSPACES||"
```
